# Desafio SCTEC IA para Devs

Aplicação web full-stack com backend e frontend para gerenciamento de empreendimentos, implementando operações CRUD completas através de API REST.

## 📋 Descrição da Solução

Esta aplicação foi desenvolvida para gerenciar empreendimentos do estado de Santa Catarina, permitindo o cadastro, listagem, edição e exclusão de informações sobre negócios locais. A solução segue as melhores práticas de desenvolvimento, utilizando padrões de design modernos, código limpo e arquitetura em camadas.

### Funcionalidades Principais

- **CRUD Completo**: Criar, ler, atualizar e deletar empreendimentos
- **Validação de Dados**: Validação tanto no frontend quanto no backend
- **Interface Moderna**: UI responsiva e intuitiva
- **API RESTful**: Endpoints bem estruturados seguindo padrões REST
- **Testes**: Cobertura de testes unitários e de integração
- **Tratamento de Erros**: Sistema robusto de tratamento de erros

### Modelo de Dados

A aplicação gerencia empreendimentos com os seguintes campos:

- **Nome do Empreendimento**: Nome do negócio (obrigatório, máx. 200 caracteres)
- **Nome do(a) Empreendedor(a)**: Nome do responsável (obrigatório, máx. 200 caracteres)
- **Município de Santa Catarina**: Cidade onde o negócio está localizado (obrigatório, máx. 100 caracteres)
- **Segmento de Atuação**: Categoria do negócio (obrigatório)
  - Tecnologia
  - Comércio
  - Indústria
  - Serviços
  - Agronegócio
- **E-mail ou Meio de Contato**: Informação de contato (obrigatório, máx. 200 caracteres)
- **Status**: Estado do empreendimento (ativo ou inativo, padrão: ativo)

### Padrões de Design Implementados

1. **Repository Pattern**: Separação da lógica de acesso a dados
2. **Service Layer**: Lógica de negócio isolada dos controladores
3. **DTO (Data Transfer Object)**: Objetos para transferência de dados
4. **Dependency Injection**: Injeção de dependências para melhor testabilidade
5. **Error Handling Middleware**: Tratamento centralizado de erros
6. **Validation Middleware**: Validação de dados reutilizável

## 🚀 Como Executar

### Pré-requisitos

- Node.js (versão 18 ou superior)
- npm (versão 9 ou superior)
- MongoDB (versão 6 ou superior) instalado e rodando localmente, ou uma instância MongoDB remota

### Instalação

1. Clone o repositório:
```bash
git clone <url-do-repositorio>
cd desafioscteciaparadevs
```

2. Instale as dependências do projeto raiz:
```bash
npm install
```

3. Instale as dependências do backend:
```bash
cd backend
npm install
```

4. Instale as dependências do frontend:
```bash
cd ../frontend
npm install
```

### Configuração

1. Configure as variáveis de ambiente do backend:
```bash
cd backend
cp env.example .env
```

Edite o arquivo `.env` com suas configurações:
```env
NODE_ENV=development
PORT=3000
MONGODB_URI=mongodb://localhost:27017/desafioscteciaparadevs
CORS_ORIGIN=http://localhost:5173
```

**Nota**: Se estiver usando MongoDB Atlas ou outra instância remota, atualize `MONGODB_URI` com a string de conexão apropriada.

### Executar em Desenvolvimento

#### Opção 1: Executar backend e frontend separadamente

Terminal 1 - Backend:
```bash
cd backend
npm run dev
```

Terminal 2 - Frontend:
```bash
cd frontend
npm run dev
```

#### Opção 2: Executar ambos simultaneamente (recomendado)

No diretório raiz:
```bash

```

A aplicação estará disponível em:
- **Frontend**: http://localhost:5173
- **Backend API**: http://localhost:3000
- **Health Check**: http://localhost:3000/health

#### Testes do Frontend

## 📦 Como Publicar/Deploy

### Opção 1: Deploy Manual

#### Backend

1. Build do backend:
```bash
cd backend
npm run build
```

2. Configure as variáveis de ambiente de produção no servidor:
```env
NODE_ENV=production
PORT=3000
MONGODB_URI=mongodb://sua-uri-de-producao
CORS_ORIGIN=https://seu-frontend.com
```

3. Execute o backend:
```bash
npm start
```

**Recomendação**: Use um process manager como PM2:
```bash
npm install -g pm2
pm2 start dist/index.js --name backend
pm2 save
pm2 startup
```

#### Frontend

1. Build do frontend:
```bash
cd frontend
npm run build
```

2. Os arquivos em `frontend/dist/` podem ser servidos por:
   - **Nginx**: Configure um servidor web para servir os arquivos estáticos
   - **Apache**: Similar ao Nginx
   - **Vercel/Netlify**: Deploy direto via CLI ou interface web
   - **AWS S3 + CloudFront**: Para hospedagem estática

**Exemplo de configuração Nginx**:
```nginx
server {
    listen 80;
    server_name seu-dominio.com;
    root /caminho/para/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

### Opção 2: Deploy com Docker (Recomendado)

#### Criar Dockerfile para Backend

Crie `backend/Dockerfile`:
```dockerfile
FROM node:18-alpine

WORKDIR /app

COPY package*.json ./
RUN npm ci --only=production

COPY . .
RUN npm run build

EXPOSE 3000

CMD ["node", "dist/index.js"]
```

#### Criar Dockerfile para Frontend

Crie `frontend/Dockerfile`:
```dockerfile
FROM node:18-alpine as build

WORKDIR /app

COPY package*.json ./
RUN npm ci

COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

#### Criar docker-compose.yml

No diretório raiz:
```yaml
version: '3.8'

services:
  mongodb:
    image: mongo:6
    container_name: mongodb
    ports:
      - "27017:27017"
    volumes:
      - mongodb_data:/data/db
    environment:
      MONGO_INITDB_DATABASE: desafioscteciaparadevs

  backend:
    build: ./backend
    container_name: backend
    ports:
      - "3000:3000"
    environment:
      NODE_ENV: production
      PORT: 3000
      MONGODB_URI: mongodb://mongodb:27017/desafioscteciaparadevs
      CORS_ORIGIN: http://localhost
    depends_on:
      - mongodb

  frontend:
    build: ./frontend
    container_name: frontend
    ports:
      - "80:80"
    depends_on:
      - backend

volumes:
  mongodb_data:
```

#### Deploy com Docker

```bash
docker-compose up -d
```

## 📝 Endpoints da API

### Base URL
```
http://localhost:3000/api
```

### Endpoints

#### GET /empreendimentos
Lista todos os empreendimentos.

**Resposta**:
```json
{
  "success": true,
  "data": [...],
  "count": 10
}
```

#### GET /empreendimentos/:id
Retorna um empreendimento específico.

**Resposta**:
```json
{
  "success": true,
  "data": {
    "_id": "...",
    "nomeEmpreendimento": "...",
    ...
  }
}
```

#### POST /empreendimentos
Cria um novo empreendimento.

**Body**:
```json
{
  "nomeEmpreendimento": "Tech Solutions",
  "nomeEmpreendedor": "João Silva",
  "municipio": "Florianópolis",
  "segmento": "Tecnologia",
  "emailContato": "joao@tech.com",
  "status": "ativo"
}
```

#### PUT /empreendimentos/:id
Atualiza um empreendimento existente.

**Body**: (campos opcionais)
```json
{
  "nomeEmpreendimento": "Tech Solutions Updated",
  "status": "inativo"
}
```

#### DELETE /empreendimentos/:id
Deleta um empreendimento.

**Resposta**:
```json
{
  "success": true,
  "message": "Empreendimento deletado com sucesso"
}
```
